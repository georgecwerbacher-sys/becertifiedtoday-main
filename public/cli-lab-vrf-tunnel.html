VRF Configuration Sim

Device in prompt:

R10 and R20 are preconfigured with the CORP VRF and reachability over the ISP underlay. Complete the missing pieces so CORP traffic between VLAN 100 (10.100.1.0/24) and VLAN 101 (10.101.2.0/24) uses Tunnel0 only. Use the R10 console first; the R20 console unlocks when R10 is complete. Under the topology, choose Device in prompt, then use the open button for that device.

Goals
  1. Extend CORP between R10 and R20 using Tunnel0 (destination = peer underlay address).
  2. Protect Tunnel0 with the preconfigured IPsec profile MyProfile.
  3. Add VRF static routes so VLAN 100 and VLAN 101 subnets reach each other via the tunnel (next-hop = peer tunnel IP). Only Tunnel0 should carry inter-site CORP traffic.

Underlay addresses used here: R10 e0/1 = 10.10.1.1, R20 e0/2 = 10.10.2.1. If you apply vrf forwarding CORP and the tunnel IP drops, re-enter the tunnel IPv4 address as in the solution.

Lab complete. Tunnel0 is extended in VRF CORP with IPsec protection, and static routes point remote CORP subnets to the peer tunnel address.
Show command sequence (spoiler)

R10

  1. conf t
  2. interface tunnel0
  3. tunnel source Ethernet0/1
  4. tunnel destination 10.10.2.1
  5. vrf forwarding CORP
  6. ip address 10.100.100.1 255.255.255.0
  7. tunnel protection ipsec profile MyProfile
  8. exit
  9. ip route vrf CORP 10.101.2.0 255.255.255.0 10.100.100.2
  10. end

R20

  1. conf t
  2. interface tunnel0
  3. tunnel source Ethernet0/2
  4. tunnel destination 10.10.1.1
  5. vrf forwarding CORP
  6. ip address 10.100.100.2 255.255.255.0
  7. tunnel protection ipsec profile MyProfile
  8. exit
  9. ip route vrf CORP 10.100.1.0 255.255.255.0 10.100.100.1
  10. end

SW20 — configuration

hostname Sw20 vlan 101 exit ! interface e0/0 switchport trunk encapsulation dot1q switchport mode trunk no shut ! interface e0/1 switchport mode access switchport access vlan 101 no shut

SW10 — configuration

hostname Sw10 vlan 100 exit ! interface e0/0 switchport trunk encapsulation dot1q switchport mode trunk no shut ! interface e0/1 switchport mode access switchport access vlan 100 no shut

ISP — configuration

hostname ISP ! interface e0/1 ip address 10.10.1.2 255.255.255.0 ip ospf 100 area 0.0.0.0 no shut ! interface e0/2 ip address 10.10.2.2 255.255.255.0 ip ospf 100 area 0.0.0.0 no shut ! router ospf 100 router-id 1.1.1.1
