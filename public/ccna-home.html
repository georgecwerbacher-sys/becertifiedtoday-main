Match your prep to how Cisco weights CCNA 200-301

The live exam follows Cisco’s published domains. CCNA 200-301 v1.1 groups content into six areas: network fundamentals, network access, IP connectivity, IP services, security fundamentals, and automation and programmability. Official guidance puts the heaviest emphasis on IP connectivity, with network fundamentals and network access close behind—those three carry most of what you see on routing, switching, and troubleshooting-style items (exact percentages can shift slightly between blueprint updates).

Approximate weights below come from Cisco exam guidance—use them to budget study hours, not as a guarantee of how many questions you will see on a specific attempt.

Domain Weight What to study
Network fundamentals 20% OSI and TCP/IP models, IPv4/IPv6, cabling, devices, topologies, subnetting basics
Network access 20% VLANs, trunking, basic STP, EtherChannel, wireless concepts, switch port configuration
IP connectivity 25% Routing tables, static routing, OSPF fundamentals, path selection, troubleshooting routing
IP services 10% NAT, DHCP, DNS, NTP, SNMP, syslog, QoS basics
Security fundamentals 15% Device hardening, ACLs, authentication, AAA basics, secure management access
Automation and programmability 10% APIs, controllers, automation concepts, JSON, Python basics, intent-based networking

What to focus on first: network fundamentals and IP connectivity underpin subnetting, routing, and a large share of drag-and-drop or simulation-style prompts. Add network access and security fundamentals next—they show up on both theory and configuration-style questions.

Suggested study order

  1. Network fundamentals
  2. Subnetting and VLSM
  3. Switching and VLANs
  4. Routing and OSPF
  5. Services (NAT, DHCP, NTP)
  6. Security and automation basics

Under time pressure, leaning hardest into IP connectivity and network fundamentals before you polish lighter domains mirrors where most candidates lose or gain points.
