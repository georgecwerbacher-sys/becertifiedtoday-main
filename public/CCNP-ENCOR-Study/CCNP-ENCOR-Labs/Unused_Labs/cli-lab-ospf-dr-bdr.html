OSPF DR/BDR Sim

Device in prompt:
Tasks

BeCertifiedToday.com site engineers inherited a partially configured OSPF deployment on these devices. Complete the remaining configuration to achieve these results:

  1. Configure R3 so that R20 is always designated as the BDR.
  2. SW10 side (Task 2): on R10, toward the SW10 uplink, set OSPF priority 0 so R10 does not participate in the DR/BDR election. Do not use the ip ospf network point-to-point command under the interface configuration to accomplish this task.
Lab complete. R3 should win DR over R20 (BDR) on 10.2.203.0/24; R10 is DROTHER with priority 0; configs saved on R3, R20, and R10.

R3

  1. R3#conf t
  2. R3(config)#int e0/1
  3. R3(config-if)#ip ospf priority 255
  4. R3(config-if)#end

R20

  1. R20#clear ip ospf process
  2. Reset all OSPF processes? [no]: y
  3. R20#conf t
  4. R20(config)#int e0/0
  5. R20(config-if)#ip address 10.2.203.20 255.255.255.0
  6. R20(config-if)#ip ospf priority 254
  7. R20(config-if)#exit
  8. R20(config)#router ospf 1
  9. R20(config-router)#network 10.2.203.0 0.0.0.255 area 20
  10. R20(config-router)#end

SW10 side (R10)

  1. R10#conf t
  2. R10(config)#int e0/0
  3. R10(config-if)#ip ospf priority 0
  4. R10(config-if)#exit
  5. R10(config)#end

Save

  1. R3# copy running-config startup-config
  2. R20# copy running-config startup-config
  3. R10# copy running-config startup-config

Helper File

  • Configure R3 so that R20 is always designated as the BDR.
  • Set R3 as DR with the highest OSPF priority so R20 will be BDR.
  • Reset the election by clearing OSPF on R20 (not R3).

R3

R3#conf t
R3(config)#int e0/1
R3(config-if)#ip ospf priority 255
R3(config-if)#end

R20

R20#clear ip ospf process
Reset all OSPF processes? [no]: y
R20#conf t
R20(config)#int e0/0
R20(config-if)#ip address 10.2.203.20 255.255.255.0
R20(config-if)#ip ospf priority 254
R20(config-if)#exit
R20(config)#router ospf 1
R20(config-router)#network 10.2.203.0 0.0.0.255 area 20
R20(config-router)#end

R20 Initial Config

interface Ethernet0/0
 ip address 10.2.203.20 255.255.255.0
 ip ospf priority 254
!
router ospf 1
 network 10.2.203.0 0.0.0.255 area 20

Task 1 Verification

  • On R3, run show ip ospf neighbor and confirm R20 is BDR.
R3#show ip ospf neighbor

Neighbor ID     Pri   State           Dead Time   Address         Interface
10.0.3.1          1   FULL/DR         00:00:37    10.0.3.1        Ethernet0/0
10.2.203.20     254   FULL/BDR        00:00:38    10.2.203.20     Ethernet0/1

Task 2 — SW10 side

  • Task 2 is entered on R10 (interface toward SW10).
  • Configure R10 so it does not participate in DR/BDR election.
  • Do not use ip ospf network point-to-point.
  • Set OSPF priority to 0 on the uplink to SW10.

SW10 side (on R10)

R10#conf t
R10(config)#int e0/0
R10(config-if)#ip ospf priority 0
R10(config-if)#exit
R10(config)#end

R10 Initial Config

interface Ethernet0/0
 ip address 10.1.102.10 255.255.255.0
!
router ospf 1
 network 10.1.102.0 0.0.0.255 area 10

Task 2 Verification

  • On R2, run show ip ospf nei and confirm R10 is DROTHER.
R2#show ip ospf nei

Neighbor ID     Pri   State           Dead Time   Address         Interface
10.0.3.1          1   FULL/BDR        00:00:37    10.0.2.1        Ethernet0/0
10.1.102.10       0   FULL/DROTHER    00:00:36    10.1.102.10     Ethernet0/1

Save the configuration

SW10 — reference (site LAN)

Task 2 OSPF commands are entered on R10 toward this switch: R10(config)#int e0/0 then ip ospf priority 0.

hostname Sw10 vlan 100 exit ! interface e0/0 switchport trunk encapsulation dot1q switchport mode trunk no shut ! interface e0/1 switchport mode access switchport access vlan 100 no shut

R10 — CLI

From R10# use show run anytime for the starting config; in config mode use do show run. When the lab reaches this step, set OSPF priority 0 on e0/0. Save last on this console.
