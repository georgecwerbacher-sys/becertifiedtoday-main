Helper File

Task 1. Using the address-family command, configure eBGP according to the topology. Use Loopback0 for the router-id.

Please check the IP address of Loopback0 interface with the command "show ip int brief" or "show run". Suppose it is 10.1.1.100 in this case.

On R1:

R1(config-router)#
router bgp 100 //please check this BGP AS number carefully with the topology, it may not be 100!
 bgp router-id 10.1.1.100
 neighbor 209.165.200.226 remote-as 200
 neighbor 209.165.202.130 remote-as 300
 address-family ipv4
  neighbor 209.165.200.226 activate
  neighbor 209.165.202.130 activate

Note: There were some reports about an error appeared when we enter "neighbor 209.165.200.226 remote-as 200" command. In that case, please try enter "address-family ipv4" and "neighbor 209.165.200.226 activate" first before typing that command.

Task 2. Advertise R1's Loopback 0, 1, and 2 networks to AS 200 and AS 300.

Note: Suppose we are still under "address-family ipv4" mode from task 1. Use "show run" to check the IP addresses and subnet masks of these loopback interfaces carefully.

On R1:

R1(config-router-af)#
R1(config-router-af)#do sh run
--output omitted--
interface loopback0
 ip address 10.1.1.100 255.255.255.255
!
interface loopback1
 ip address 209.165.201.1 255.255.255.248
!
interface loopback2
 ip address 209.165.201.9 255.255.255.248
!

With BGP, we have to advertise the networks with the exact subnet masks which are configured on each interface. Therefore:

  • with loopback 0 it should be "10.1.1.100 mask 255.255.255.255"
  • with loopback 1 it should be "209.165.201.0 mask 255.255.255.248"
  • with loopback 2 it should be "209.165.201.8 mask 255.255.255.248"

On R1, type these commands (under "address-family ipv4" mode from task 1):

R1(config-router-af)#
network 10.1.1.100 mask 255.255.255.255
network 209.165.201.0 mask 255.255.255.248
network 209.165.201.8 mask 255.255.255.248
