Task 1. Modify the existing ACL on R10 so that EIGRP routes are received from R20 and R30

  • The modification should only allow EIGRP routes to pass.
  • Do not remove any configuration from R10.

Check EIGRP neighbors first

On R10, run show ip eigrp neighbors:

R10

R10#show ip eigrp neighbors
EIGRP-IPv4 Neighbors for AS(10)
(Empty)
  • R10 has no EIGRP neighbors currently because ACL 150 is blocking EIGRP.
  • EIGRP is not TCP, UDP, or ICMP, so it is filtered out.

Inspect ACL 150

Check sequence numbers with show ip access-list 150:

R10

R10#show ip access-list 150
Extended IP access list 150
    10 permit tcp any any
    20 permit udp any any
    30 permit icmp any any
    40 deny ip any any (186 matches)
  • This ACL blocks EIGRP from being received on R10.
  • Permit all EIGRP-related packets before sequence 40 (the deny all). In this case use sequence 5:

On R10

R10(config)#ip access-list extended 150
R10(config-ext-nacl)#5 permit eigrp any any

Verification

Check EIGRP neighbors again on R10:

R10

R10#show ip eigrp neighbors 
EIGRP-IPv4 Neighbors for AS(10)
H   Address                 Interface              Hold Uptime   SRTT   RTO  Q  Seq
                                                   (sec)         (ms)       Cnt Num
1   192.168.123.20          Et0/0                    12 00:02:11   10   100  0  9
0   192.168.123.30          Et0/0                    14 00:02:14   10   100  0  10

R20 (192.168.123.20) and R30 (192.168.123.30) are now EIGRP neighbors on R10.

Task 2. Configure CoPP on R20

  • Permit ICMP traffic from 192.168.24.0/24.
  • Limit traffic to 8,000 bps.
  • Discard additional packets.
  • Note: Read the requirement carefully—it may ask for SSH or Telnet instead of ICMP.
  • Example below uses ICMP: access-list 100 permit icmp 192.168.24.0 0.0.0.255 any. For Telnet or SSH, use the matching TCP permit (for example port 23 or 22) per the task wording; keep the class-map / policy-map / police shape unless the lab specifies different names or rates.

On R20

access-list 100 permit icmp 192.168.24.0 0.0.0.255 any
!
class-map CoPP_ICMP
 match access-group 100
!
policy-map CHECK_ICMP
 class CoPP_ICMP
  police 8000 
   conform-action transmit 
   exceed-action drop
!
control-plane 
 service-policy input CHECK_ICMP

Version 3 — SSH (not ICMP). Configure CoPP on R20

  • Permit SSH traffic from 192.168.24.0/24 (not ICMP).
  • Limit traffic to 10,000 bps.
  • Discard additional packets.

On R20

access-list 100 permit tcp 192.168.24.0 0.0.0.255 any eq 22
!
class-map CoPP_SSH
 match access-group 100
!
policy-map CHECK_SSH
 class CoPP_SSH
  police 10000 
   conform-action transmit 
   exceed-action drop 
!
control-plane
 service-policy input CHECK_SSH

Use Open R10 CLI or Open R20 CLI (the button switches when the R20 CoPP phase unlocks).
