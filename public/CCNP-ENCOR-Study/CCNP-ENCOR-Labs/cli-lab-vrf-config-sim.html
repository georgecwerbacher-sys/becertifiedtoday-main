Tasks

BeCertifiedToday engineers were tasked to support a new site rollout. Site personnel began configuring network devices, and R10 and R20 are preconfigured with the CORP VRF. R10 has network connectivity to R20. Complete the configurations to achieve these goals:

  1. Extend the CORP VRF between R10 and R20 using Tunnel0.
  2. (Optional) Protect Tunnel0 using the preconfigured profile.
  3. Configure static routing on R10 and R20 so that users in VLAN 100 and VLAN 101 that belong to the CORP VRF are able to communicate with each other. Tunnel0 should be the only interface used to route traffic for the CORP VRF.
Lab complete. Tunnel0 is extended in VRF CORP with IPsec protection, and static routes point remote CORP subnets to the peer tunnel address.

R10

  1. conf t
  2. interface tunnel0
  3. tunnel source Ethernet0/1 (or e0/1)
  4. tunnel destination 10.10.2.1
  5. vrf forwarding CORP
  6. ip address 10.100.100.1 255.255.255.0
  7. exit (leave tunnel interface)
  8. interface tunnel0 (re-enter Tunnel0)
  9. tunnel protection ipsec profile MyProfile
  10. exit
  11. ip route vrf CORP 10.101.2.0 255.255.255.0 10.100.100.2 or ip route vrf CORP 10.101.2.0 255.255.255.0 Tunnel0
  12. end

R20

  1. conf t
  2. interface tunnel0
  3. tunnel source Ethernet0/2 (or e0/2)
  4. tunnel destination 10.10.1.1
  5. vrf forwarding CORP
  6. ip address 10.100.100.2 255.255.255.0
  7. exit (leave tunnel interface)
  8. interface tunnel0 (re-enter Tunnel0)
  9. tunnel protection ipsec profile MyProfile
  10. exit
  11. ip route vrf CORP 10.100.1.0 255.255.255.0 10.100.100.1 or ip route vrf CORP 10.100.1.0 255.255.255.0 Tunnel0
  12. end

Helper File

  • Run show ip interface brief on R10 and R20 to confirm underlay IPs for tunnel destination.
  • Use R10 e0/1 IP and R20 e0/2 IP as tunnel destinations (for example 10.10.1.1 and 10.10.2.1).
  • Retyping tunnel source, vrf forwarding CORP, and tunnel IP commands is expected to ensure they are present.
  • When vrf forwarding CORP is applied, interface IPv4 settings are cleared and must be reconfigured.

R10

R10(config)#interface tunnel 0
R10(config-if)#tunnel source e0/1
R10(config-if)#tunnel destination 10.10.2.1
R10(config-if)#vrf forwarding CORP
R10(config-if)#ip address 10.100.100.1 255.255.255.0

R20

R20(config)#interface tunnel 0
R20(config-if)#tunnel source e0/2
R20(config-if)#tunnel destination 10.10.1.1
R20(config-if)#vrf forwarding CORP
R20(config-if)#ip address 10.100.100.2 255.255.255.0

Part 2 - Tunnel protection

  • Protect Tunnel0 using the preconfigured profile.
  • Task 2 may or may not exist in the prompt, so verify the requirement before configuring it.
  • Use show run to find the preconfigured IPsec profile name (for example MyProfile).

R10 and R20

R10(config)#interface tunnel 0
R10(config-if)#tunnel protection ipsec profile MyProfile
R20(config)#interface tunnel 0
R20(config-if)#tunnel protection ipsec profile MyProfile

Part 3 - Static routing and verification

  • After tunnel IPs work, type exit once, then interface tunnel0 again before tunnel protection ipsec profile MyProfile (simulator matches that order).
  • Static route: use the peer tunnel IP as next hop, or use egress Tunnel0 — for example ip route vrf CORP 10.101.2.0 255.255.255.0 Tunnel0 on R10 (either form is accepted).
  • After end on both routers, save and Part 3 pings can be done in either order (simulator accepts both).

Static routes and save

R10(config)#ip route vrf CORP 10.101.2.0 255.255.255.0 Tunnel0
R20(config)#ip route vrf CORP 10.100.1.0 255.255.255.0 Tunnel0
R10#copy running-config startup-config
R20#copy running-config startup-config
