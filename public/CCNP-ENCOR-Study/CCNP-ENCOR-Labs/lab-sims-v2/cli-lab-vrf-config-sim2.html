Task 1. Extend the Finance VRF between R11 and R22 using Tunnel 10.

We need to create the VRF Finance and assign the interface tunnel 10 to it.

On R22:

R22(config)#
vrf definition Finance //Old command: ip vrf Finance
rd 100:1 //we can use any number or ignore this command
address-family ipv4 //Enable the IPv4 address family within the VRF, required by new IOS versions
exit
interface Tunnel10
vrf forwarding Finance //old IOS versions require the "ip vrf forwarding Finance" instead
ip address 10.10.10.2 255.255.255.252
tunnel source e0/0
tunnel destination 209.165.200.230
no shut
exit
exit

Note: This router is using a modern IOS version so old command "ip vrf Finance" (to create this VRF) does not work, we have to use the "vrf definition Finance" command instead. We may need to use the "address-family ipv4" command under VRF to enable IPv4 address family.

Task 2. Complete the Finance VRF configuration on R22 and configure static routing so that traffic between VLAN 111 and VLAN 222 uses Tunnel 10 exclusively.

To extend the VRF Finance to Sw22, we need to assign e0/1 interface of R22 to VRF Finance as well:

On R22:

R22(config)#
interface e0/1
vrf forwarding Finance
//We will see this notification: "% Interface Ethernet0/1 IPv4
//disabled and address(es) removed due to enabling VRF Finance"
ip address 10.22.22.1 255.255.255.252 //reassign IP address

Configure static routing on R22:

R22(config)#
ip route vrf Finance 10.10.111.0 255.255.255.0 tunnel10
ip route vrf Finance 10.10.222.0 255.255.255.0 10.22.22.2

Next:

R22(config)#
interface e0/1
vrf forwarding Finance
ip address 10.22.22.1 255.255.255.252
ip route vrf Finance 10.10.111.0 255.255.255.0 tunnel10
ip route vrf Finance 10.10.222.0 255.255.255.0 10.22.22.2
