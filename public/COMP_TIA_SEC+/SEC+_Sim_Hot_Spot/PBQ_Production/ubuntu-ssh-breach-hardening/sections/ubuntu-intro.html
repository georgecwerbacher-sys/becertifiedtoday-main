Incident response · SY0-701 PBQ

Ubuntu 22.04 web server — post-breach hardening

A threat actor gained SSH access using brute-forced credentials. Forensics shows the attacker pivoted to root via a misconfigured sudo policy and established persistence with a malicious cron job. Harden all four areas below — some fixes interact. Wrong changes in one file can break another control.
