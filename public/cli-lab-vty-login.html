VTY Login Sim

You are consoled into R1. The topology is shown above. Configure R1 so the local user can reach privilege 15, VTYs use the local database with Telnet and SSH, exec sessions time out after 5 minutes of inactivity, then save the configuration. (This simulator only checks R1 commands; real Telnet from R2/R3 would use the verification steps at the bottom.)

Protect R1 — three results:
  1. Local user SuperAdmin / SuperUser with highest privilege (15). secret or password is accepted.
  2. VTY 0–4: login local, transport input telnet ssh.
  3. exec-timeout 5 0 (5 minutes, 0 seconds).
Finish: return to privileged EXEC and run copy running-config startup-config (or copy run start).
R1(config)#
Lab complete. Configuration is saved. On a live network you could verify with Telnet from R2 or R3 to 192.168.100.1 (use show ip interface brief on R1 if that differs).
