CCNA Trunking & LACP Practice Simulation

Select device

Tasks

  1. Task 1. Configure trunks between Sw1 and Sw2 on ports GigabitEthernet0/1 and GigabitEthernet0/2 using the IEEE standard frame tagging method (802.1Q).
    • Add VLAN 45 as untagged (native) on the trunk ports.
    • Only extend VLAN 15 and the untagged VLAN across the trunk.
    • Verify that PC1 can ping PC2 (ping 10.15.15.20) after Task 1 trunks are in place on both switches.
  2. Task 2. On Sw1 and Sw2, use IEEE 802.3ad link aggregation (LACP).
    • Combine Gi0/1 and Gi0/2 into a single logical link while leaving the trunk behavior on Port-channel15.
    • Assign number 15 to the link (channel-group 15 mode active).
    • Both sides must negotiate aggregation (active LACP).

Trunk + LACP checklist

  • Task 1: On each switch, 802.1Q trunks on GigabitEthernet0/1 and GigabitEthernet0/2; native VLAN 45; switchport trunk allowed vlan 15,45. The global VLAN-create lines vlan 15 and vlan 45 may be entered in either order. On each trunk (and on Port-channel15 in Task 2), native and allowed-VLAN lines may also be entered in either order. After each physical interface block, return to global config with exit.
  • Task 2: interface range GigabitEthernet0/1 - 2, channel-protocol lacp, channel-group 15 mode active, then exit; apply the same trunk settings on Port-channel15; leave configuration mode with end.
  • Run the same STEPS sequence on Sw1 and Sw2.
  • After Task 1 on both switches, from PC1 run ping 10.15.15.20.
  • From enable, optional: show interfaces trunk on a switch after that switch finishes Task 1 (same Po15-style summary on Sw1 and Sw2). show etherchannel summary after the full STEPS sequence on both switches.
  • Save on both switches from enable: copy running-configuration startup-configuration.

Solution (spoiler)

Task 1 — Sw1 (physical trunks)

Sw1 — Task 1 
Sw1# configure terminal
Sw1(config)# vlan 15
Sw1(config)# vlan 45
Sw1(config)# interface GigabitEthernet0/1
Sw1(config-if)# switchport trunk encapsulation dot1q
Sw1(config-if)# switchport mode trunk
Sw1(config-if)# switchport trunk native vlan 45
Sw1(config-if)# switchport trunk allowed vlan 15,45
Sw1(config-if)# exit
Sw1(config)# interface GigabitEthernet0/2
Sw1(config-if)# switchport trunk encapsulation dot1q
Sw1(config-if)# switchport mode trunk
Sw1(config-if)# switchport trunk native vlan 45
Sw1(config-if)# switchport trunk allowed vlan 15,45
Sw1(config-if)# exit

Verification — after Task 1

  • Finish the Task 1 command sequence on Sw1 and Sw2 (you should still be at Sw1(config)# / Sw2(config)#).
  • From PC1, run ping 10.15.15.20. The sim allows replies only after Task 1 is complete on both switches.
  • Optional: from privileged EXEC on a switch, show interfaces trunk — same Po15-style summary on Sw1 or Sw2 once that switch has finished Task 1.

Task 2 — Sw1 (LACP + Po15 trunk)

Sw1 — Task 2 
Sw1(config)# interface range GigabitEthernet0/1 - 2
Sw1(config-if-range)# channel-protocol lacp
Sw1(config-if-range)# channel-group 15 mode active
Sw1(config-if-range)# exit
Sw1(config)# interface Port-channel15
Sw1(config-if)# switchport trunk encapsulation dot1q
Sw1(config-if)# switchport mode trunk
Sw1(config-if)# switchport trunk native vlan 45
Sw1(config-if)# switchport trunk allowed vlan 15,45
Sw1(config-if)# end

Verification — after Task 2

  • Finish the Task 2 sequence on Sw1 and Sw2 through end so both return to privileged EXEC (Sw1# / Sw2#).
  • Optional: from enable, show etherchannel summary (sim output appears only after the full STEPS sequence on both switches). You can also run show interfaces trunk again.
  • Lab pass: PC1 ping must have succeeded, then from enable on each switch run copy running-configuration startup-configuration (copy run start) on Sw1 and Sw2.
