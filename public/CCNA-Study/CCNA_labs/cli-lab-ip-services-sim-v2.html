Tasks

Services must be configured in the order presented to complete the implementation.

  1. BeCertifiedToday.com engineers were tasked to support a new site rollout. On R2, configure dynamic one-to-one address mapping using a standard list named XLATE so traffic sourced from R3 can be translated into pool test_pool on the 10.10.10.0/24 network when it is sent toward R1. Do not use an NVI configuration. When you are done, confirm reachability with a ping from R3 to 192.168.100.1. The lab is complete only after that NAT work is done and you run copy running-configuration startup-configuration from privileged exec on R2, R1, and R3 (abbrev. copy run start).
  2. Task 2 (R3): from configuration mode, assign GigabitEthernet0/2 via DHCP: interface GigabitEthernet0/2 then ip address dhcp (see Spoiler for the exact prompt sequence).
  3. Task 3: On R1, use configure terminal then ntp master 1 to act as the NTP server. On R2, configure an NTP client to 10.1.2.1 (not a peer). See Spoiler for the exact R1 prompt sequence.
  4. Configure SSH access from R1 to R3, while excluding access via other remote connection protocols using the user root and password s3cret on router R3 using RSA. Verify connectivity from router R1 to R3 using a destination address assigned to interface GigabitEthernet0/2 on R3.

IP Services Sim v2 — implementation hints

  • Task 1 (R2): Build standard ACL XLATE (permit the sources R3 needs), define ip nat pool test_pool on 10.10.10.0/24, then ip nat inside source list XLATE pool test_pool without NVI. Apply ip nat inside and ip nat outside on the interfaces toward R3 and R1. From R3, ping 192.168.100.1 once routing and NAT line up.
  • Task 2 (R3): DHCP client on GigabitEthernet0/2: interface GigabitEthernet0/2, then ip address dhcp. Confirm the DHCP server (often R1) has a scope for that segment.
  • Task 3: On R1, configure terminal then ntp master 1. On R2, ntp server 10.1.2.1 so R2 is a client to R1’s address toward R2.
  • Task 4 (R3): crypto key generate rsa; user root with secret s3cret; line vty 0 4 with transport input ssh only; from R1, SSH to R3’s GigabitEthernet0/2 address as root.
  • Useful checks: on R2, show ip nat translations and show access-lists XLATE; on R3, show ip interface brief; from R1, ssh root@ plus that G0/2 IP.

Solution (spoiler)

Task 1 — R2 NAT

R2 — Terminal 
R2# configure terminal
R2(config)# ip access-list standard XLATE
R2(config-std-nacl)# permit 10.2.3.3
R2(config-std-nacl)# permit 192.168.3.1
R2(config-std-nacl)# permit 10.1.3.11
R2(config-std-nacl)# exit
R2(config)# interface GigabitEthernet0/1
R2(config-if)# ip nat inside
R2(config-if)# exit
R2(config)# interface GigabitEthernet0/0
R2(config-if)# ip nat outside
R2(config-if)# exit
R2(config)# ip nat pool test_pool 10.10.10.1 10.10.10.254 netmask 255.255.255.0
R2(config)# ip nat inside source list XLATE pool test_pool

Task 2 — R3 DHCP

R3 — Terminal 
R3# configure terminal
R3(config)# interface GigabitEthernet0/2
R3(config-if)# ip address dhcp

Task 3 — R1 NTP server

R1 — Terminal 
R1# configure terminal
R1(config)# ntp master 1

Task 4 — R3 SSH

R3 — Terminal 
R3# configure terminal
R3(config)# crypto key generate rsa modulus 1024
R3(config)# username root privilege 15 secret s3cret
R3(config)# line vty 0 4
R3(config-line)# login local
R3(config-line)# transport input ssh
R3(config-line)# end
