OSPF on R1 — helper walkthrough

Task 1. Process ID and router-ID

Configure OSPF on R1 with a process ID and router-ID only as follows:

  • use process ID 33
  • use E0/1 IP as the router ID

First use show ip interface brief on R1 to confirm the Ethernet0/1 address (this sim’s graded CLI does not implement that command; use it on real gear or read the value from show running-config / the topology). 

R1# show ip interface brief
Interface              IP-Address      OK? Method Status                Protocol
Ethernet0/0            10.0.22.1       YES Manual up                    up
Ethernet0/1            10.0.33.1       YES Manual up                    up

→ The IP address of E0/1 is 10.0.33.1, so use it for router-id.

Note: Always verify the E0/1 address in your environment; it may differ from the example.

On R1:

R1(config)# router ospf 33
R1(config-router)# router-id 10.0.33.1

Task 2. Adjacencies without network, and DR

  • Configure R1 to establish OSPF neighbor adjacencies with R2 and R3. Do not use a network statement under the OSPF process.
  • Configure R1 to always become the DR for Area 0 on its segments.

On R1 (enable OSPF on both WAN interfaces; high priority for DR):

R1(config-router)# exit
R1(config)# interface e0/0
R1(config-if)# ip ospf 33 area 0
R1(config-if)# ip ospf priority 255
R1(config-if)# interface e0/1
R1(config-if)# ip ospf 33 area 0
R1(config-if)# ip ospf priority 255

Verification

On real equipment you may clear the OSPF process so DR/BDR is re-elected after changing priority (not implemented in this sim): 

R1(config-if)# exit
R1(config)# exit
R1# clear ip ospf process
Reset ALL OSPF processes? [no]: y

Then use show ip ospf neighbor or show ip ospf interface:

R1# show ip ospf neighbor
Neighbor ID     Pri   State        Dead Time   Address      Interface
10.2.2.2          1   FULL/BDR     00:00:35    10.0.22.2    Ethernet0/0
10.3.3.3          1   FULL/BDR     00:00:35    10.0.33.3    Ethernet0/1

or

R1# show ip ospf interface
Ethernet0/0 is up, line protocol is up
  Internet address is 10.0.22.1/30, Area 0
  Process ID 33, Router ID 10.0.33.1, Network Type BROADCAST, Cost: 1
  Transmit Delay is 1 sec, State DR, Priority 255
  ...
Ethernet0/1 is up, line protocol is up
  Internet address is 10.0.33.1/29, Area 0
  Process ID 33, Router ID 10.0.33.1, Network Type BROADCAST, Cost: 1
  Transmit Delay is 1 sec, State DR, Priority 255
  ...

→ On R1, Ethernet0/0 and Ethernet0/1 should show DR with neighbors as BDR when R2/R3 use default priority.

Save the configuration

R1# copy running-config startup-config
