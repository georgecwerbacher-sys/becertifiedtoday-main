NAT, NTP, DHCP relay, and SSH — lab helper

  • Task 1 — R1: finish NAT/PAT (overload).
  • Task 2 — R1 / Sw1: NTP server on R1 (Lo0); NTP client on Sw1.
  • Task 3 — Sw1 / PC1: DHCP relay toward R1; PC1 uses DHCP.
  • Task 4 — Sw1 / PC1: RSA SSH and test from PC1.
  • Save running-config to startup on R1 and Sw1 when done.

Task 1. Complete the configuration of NAT/PAT on the R1 router using the predefined configuration.

  • Inside, outside, and the access list are already configured — add the remaining NAT/PAT command below.
  • Use PAT (overload), not static NAT, because more than one host shares the outside.
R1# configure terminal
R1(config)# ip nat inside source list 192 interface e0/0 overload

Verification

  • Finish Task 3 first so PC1 gets DHCP, then verify NAT; or use extended ping from Sw1 to 10.0.18.10 with source Ethernet0/1.
  • Confirm translations with show ip nat translations on R1.

Task 2. Configure NTP on R1 and Sw1 (NTP server = R1; source = Loopback 0; client = Sw1). 

R1# configure terminal
R1(config)# ntp master
R1(config)# ntp source-interface Loopback0
Sw1# configure terminal
Sw1(config)# ntp server 209.165.202.134
! use R1 Loopback0 address (209.165.202.134)

Verification

  • Wait about one minute, then run show ntp status on Sw1 (example output below).
  • You should see Sw1 synchronized with the NTP server on R1.
  • If the clock is still unsynchronized, run ping 209.165.202.134 from Sw1 to confirm reachability.
Sw1# show ntp status
