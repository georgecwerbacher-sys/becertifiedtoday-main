Task 1 — Local account on Sw3 (Telnet, VTY 0–4)

On Sw3, create the user, then allow Telnet only on VTY 0–4 with local login: 

Sw3# configure terminal
Sw3(config)# username devnet algorithm-type sha256 privilege 15 secret access8cli
Sw3(config)# line vty 0 4
Sw3(config-line)# transport input telnet
Sw3(config-line)# login local
Sw3(config-line)# end
Sw3#

Note: Privilege level 15 is privileged EXEC (enable mode) and allows all commands by default. You cannot use the keyword password after algorithm-type sha256; use secret instead.

Verification

Find an address on Sw3 to Telnet to (from PC1). On Sw3, use show ip interface brief. Example: if Ethernet0/2 is 172.16.1.6: 

Sw3# show ip interface brief

Interface        IP-Address      OK? Method    Status Protocol
...
Ethernet0/2      172.16.1.6      YES manual    up     up

From PC1 (your IP may differ):

PC1> telnet 172.16.1.6
Trying 172.16.1.6 ... Open
User Access Verification
Username: devnet
Password: access8cli
Sw3#

Task 2 — Extended ACL INTERNET_ACL and apply on R1

  • Allow HTTPS from 172.16.0.0/16 (TCP port 443).
  • Allow Telnet only for VLAN 101 (use 172.16.101.0/24 from the topology — confirm with show run on Sw1 for Ethernet0/0 in VLAN 101).
  • Deny other traffic with log-input (ingress interface, source MAC, source/dest IP and ports).

On R1, add ACEs to the existing named extended ACL:

R1# configure terminal
R1(config)# ip access-list extended INTERNET_ACL
R1(config-ext-nacl)# permit tcp 172.16.0.0 0.0.255.255 any eq 443
R1(config-ext-nacl)# permit tcp 172.16.101.0 0.0.0.255 any eq telnet
R1(config-ext-nacl)# deny ip any any log-input
R1(config-ext-nacl)# exit

Apply inbound on both LAN-facing interfaces (Ethernet0/0 and Ethernet0/3): 

R1(config)# interface Ethernet0/0
R1(config-if)# ip access-group INTERNET_ACL in
R1(config-if)# exit
R1(config)# interface Ethernet0/3
R1(config-if)# ip access-group INTERNET_ACL in

log-input vs log: log logs a basic match. log-input adds ingress interface, source MAC, and source/destination IPs and ports. Use log-input here for the required detail.

Task 3 — DHCP snooping on Sw1

  • Enable DHCP snooping for VLAN 101.
  • Disable DHCP Option-82 data insertion.
  • Enable DHCP snooping MAC address verification.
Sw1(config)# ip dhcp snooping
Sw1(config)# ip dhcp snooping vlan 101
Sw1(config)# no ip dhcp snooping information option
Sw1(config)# ip dhcp snooping verify mac-address
