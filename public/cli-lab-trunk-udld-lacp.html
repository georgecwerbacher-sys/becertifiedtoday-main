Trunk, UDLD & LACP Sim

Configure SW10 only (no access to SW20 or SW30). Restrict trunk allowed VLANs to VLANs that are in use. Use PC2 to verify reachability when that console unlocks. STP may take up to about a minute before pings succeed (simulated as immediate). Under the topology, choose Device in prompt, then the open button for that device.

Tasks
  1. Fix the trunk toward SW30 on E0/0 so PC2 can ping PC1 at 10.10.100.10.
  2. Enable aggressive UDLD on SW10 E0/0.
  3. Fix LACP negotiation for the port-channel to SW20 (E0/2–E0/3) so PC2 can ping PC3 at 10.10.100.30.
  4. (Optional) On E0/1, allow forwarding as soon as the link is up (access VLAN 300).
  5. (Optional) Use 32-bit spanning-tree path cost values on SW10.
  6. Save the configuration on SW10.
Device in prompt:
Lab complete. E0/0 is a dot1q trunk (VLAN 300) with UDLD aggressive; Port-channel 10 LACP active with trunks allowing 1,300; optional portfast and long pathcost; config saved.
Command sequence (spoiler)

Task 1 — E0/0 to SW30

  1. conf t → int e0/0 → switchport trunk encapsulation dot1q → switchport mode trunk → switchport trunk allowed vlan 300 → end

Task 2 — UDLD

  1. conf t → int e0/0 → udld port aggressive → end

Task 3 — LACP

  1. conf t → no interface po10 (or no interface port-channel10)
  2. interface range e0/2 - 3 → dot1q trunk → switchport trunk allowed vlan 1,300 → channel-group 10 mode active → end

Optional: int e0/1 → switchport mode access → spanning-tree portfast; spanning-tree pathcost method long (global).

Save: copy running-config startup-config
